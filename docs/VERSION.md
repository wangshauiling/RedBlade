# 文档版本管理规范

## 一、版本号规则

采用三段式版本号：`v主版本号.次版本号.修订号`

| 版本段 | 说明 | 示例 |
|--------|------|------|
| 主版本号 | 重大架构调整或重构 | v1.0.0 → v2.0.0 |
| 次版本号 | 新增功能或模块 | v1.0.0 → v1.1.0 |
| 修订号 | 文档修正、补充说明 | v1.0.0 → v1.0.1 |

## 二、版本记录格式

每个文档开头必须包含版本记录表格：

```markdown
## 文档版本记录

| 版本号 | 修改日期 | 修改人 | 修改内容 |
|--------|----------|--------|----------|
| v1.0.0 | 2026-05-20 | 王帅令 | 初始版本，创建文档 |
| v1.1.0 | 2026-05-20 | 王帅令 | 新增：xxx功能；补充：xxx说明 |
```

## 三、修改内容描述规范

### 3.1 新增内容

格式：`新增：功能名称`

示例：
- `新增：在线用户管理架构`
- `新增：Druid数据库监控配置`

### 3.2 补充内容

格式：`补充：补充说明`

示例：
- `补充：Token刷新机制详细说明`
- `补充：登录失败限制机制`

### 3.3 修正内容

格式：`修正：修正说明`

示例：
- `修正：API路径错误`
- `修正：配置项描述不准确`

### 3.4 删除内容

格式：`删除：删除说明`

示例：
- `删除：已废弃的接口`
- `删除：过时的配置项`

### 3.5 多项修改

使用分号分隔：

示例：`新增：xxx功能；补充：xxx说明；修正：xxx错误`
